# Resume Bullet Quality Rules

## Good Bullet Pattern

```text
[Action verb] [specific work] across [scope] using [tools], resulting in [verified outcome].
```

## Good Examples

```text
Built an internal automation workflow that reduced manual review time by 40% by standardizing evidence collection and report assembly.

Created Python scripts to process vulnerability-export data, helping engineering teams prioritize high-risk findings more quickly.

Implemented policy-as-code controls across GKE clusters using OPA and admission controllers.

Tuned SIEM detection rules to cut phishing false positives by 30%.
```

## Bad Examples

```text
Responsible for security operations.
Worked on various automation projects.
Used multiple tools to improve outcomes.
Passionate about technology and innovation.
Leveraged robust solutions to drive dynamic outcomes.
```

## Rules

- No vague filler.
- No unsupported metrics. If you cannot prove the number, do not use it.
- No unsupported tools. Only list tools you actually used.
- No fake leadership scope. "Led" means you led. "Contributed to" is honest.
- No invented certifications.
- If the user cannot support a claim, soften it or omit it.

## Evidence Test

For every bullet, ask:

1. What did the person actually do?
2. What changed because of it?
3. What tools or systems were involved?
4. What scale was involved?
5. Is the result verified?
6. Is this relevant to the target job?
7. Could a recruiter understand it quickly?
8. Could a technical interviewer challenge it?

If the bullet fails any of these, rewrite it. If it cannot be fixed, remove it.

## Verification Checklist

- [ ] Every bullet traces to a database entry or user answer.
- [ ] No metric appears without a source.
- [ ] No tool appears that is not in the tool inventory or user answer.
- [ ] No date appears without confirmation.
- [ ] No leadership claim without evidence of leadership.
- [ ] The bullet is relevant to the target job.
- [ ] The bullet is defensible in an interview.
